About Section

Enhancing Accessibility Awareness through Bootlegging Application

Introduction:

Innovation and creativity are pivotal for societal progress, yet many individuals struggle with finding inspiration and generating ideas. This difficulty is compounded by a lack of perspective-taking, particularly regarding the experiences of individuals with accessibility needs. This project aims to address this challenge by developing an application to facilitate idea sparking, formerly known as bootlegging, and promote inclusivity in creative thinking, with a focus on catering to the needs of individuals with disabilities.

Problem Statement:

As a society, we often face difficulty in empathetically immersing ourselves in the experiences of others, particularly those with accessibility issues. This lack of perspective-taking leads to a failure to consider the needs and perspectives of individuals with disabilities in the design and development of technology. Consequently, the majority overlooks the importance of including features that cater to the accessibility needs of a smaller population, resulting in a lack of inclusivity and oversight in future technology.

Relevance to Accessibility:

Accessibility awareness is essential for ensuring that technology and innovation are inclusive and equitable for all individuals, regardless of ability. By developing an application focused on bootlegging and creative thinking, specifically tailored to the future benefit of accommodating accessibility needs, we can promote awareness and understanding of the challenges faced by individuals with disabilities. Through inclusivity in the creative process, we aim to foster empathy and advocate for the integration of accessible features in future technologies.

Main Objectives:

  1. Create an application to enhance creativity:

    We aim to develop an application that assists individuals in generating ideas and fostering creativity. Our primary focus is to ensure the accessibility of this application, making it usable for people of all abilities, including those with disabilities.

  2. Enhance accessibility features:

    We will incorporate additional features into the application to facilitate its use by individuals with disabilities. These enhancements aim to eliminate any obstacles that may hinder their participation in creative endeavors.

  3. Utilize interactive experiences for education:

    Interactive experiences within the application will be utilized to educate users about the challenges faced by individuals with disabilities. This educational approach seeks to foster a deeper understanding of these challenges and promote empathy among users.

  4. Promote accessibility in future technologies:

    We will advocate for the integration of accessibility considerations in the design of future technologies. By prioritizing accessibility, we aim to contribute to a more inclusive tech industry, ensuring that individuals of all abilities can actively participate in the creative process.

Altogether, our aim is to create a more inclusive and fair environment where everyone can share their ideas and perspectives.

Plan for This Semester:

  • Semester Focus:

    Developing an accessible bootlegging application using HTML, JavaScript, and CSS.

  • Primary Goal:

    Ensure inclusivity by implementing a range of accessibility features.

  • Integration:

    - Integrate a randomizer word generator to aid users in sparking creative ideas.
    - Design the application's interface and functionality with a focus on simplicity and ease of use.

  • Accessibility Enhancement:

    - Implement semantic HTML elements, ARIA roles, and keyboard navigation.
    - Incorporate options for adjusting website settings to accommodate users with various accessibility needs, such as low vision and color blindness.

  • Testing:

    - Thoroughly test the application using multiple accessibility checkers.
    - Conduct user testing with assistive technologies like screen readers.

  • Objective:

    Prioritize accessibility at every stage of development to create an application that is usable by everyone and quick and intuitive for those relying on assistive technologies.

Use Case Analysis of Scenarios:

Viewers:

Story: As a viewer, I want to be able to access any relevant pages on the bootlegging idea website so that I can click on multiple generators to get a random string of ideas and possible ideas.

  • A viewer can view all pages on the website.
  • A viewer can click on multiple/single generators to obtain a random word(s) for that specific topic.
  • A viewer can input a word into the generator.
  • A viewer can view a concatenated string output from the randomized generator.

    • Use Cases:

    Viewer Use Case:This is an overview of what a viewer can do using our web application.

    Use Case: UV1 View all webpages

    Description:The system allows a viewer to view and navigate all web pages on the web application.

    Actor(s):Viewer

    Triggering Event:The user navigates to a different page using the navigation panel.

    Stakeholders:N/A

    PreconditionsThe user must have clicked on a page to navigate to on the navigation panel.

    Post Conditions:The correctly chosen web page appears.


    Use Case: UV2 Click on generator(s)

    Description:The system allows a viewer to click on the various generators to gain the output of a randomized word related to that generator topic.

    Actor(s):Viewer

    Triggering Event:The user clicks on one generator or the randomize all generators button.

    Stakeholders:N/A

    PreconditionsThe user must have clicked on the button to run one generator button or all the generators button.

    Post Conditions:A word comes as an output for each specified generator.


    Use Case: UV3 Input word into generator(s)

    Description:The system allows a viewer to input/add word(s) into the generator(s) database.

    Actor(s):Viewer

    Triggering Event:The user clicks on add a word to a specific generator and inputs a word regarding the generator's topic.

    Stakeholders:N/A

    PreconditionsThe user must have clicked on the add button to a specific generator and input a word.

    Post Conditions:An alert pops up saying that the word is added to that generator’s database.


    Use Case: UV4 View generated concatenated string

    Description:The system allows a viewer to generate a concatenated string by running all of the generators.

    Actor(s):Viewer

    Triggering Event:The user clicks on all of the generator's randomized buttons or the randomized all button.

    Stakeholders:N/A

    PreconditionsThe user must have clicked on all of the generator’s randomized buttons or the randomized all button.

    Post Conditions:The concatenated string of words appears as the output.

    Conclusion:

    The proposed project seeks to address the lack of perspective-taking and empathy in society regarding the needs of individuals with disabilities by developing an innovative application focused on idea sparking and accessibility awareness. By promoting inclusivity and empathy in the creative process, we aim to advocate for the integration of accessible design principles in future technologies, ensuring equitable access and participation for all individuals.
